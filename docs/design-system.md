# オシドリ デザイン設計書

## 1. ブランドコンセプト

### 1.1 サービスの世界観

**オシドリ**は、飲食店の「想い」と消費者の「推し」をつなぐプラットフォームである。

名前の由来は、仲睦まじい鳥「おしどり」と、誰かを応援する「推し」の文化を掛け合わせたもの。飲食店とファンが互いに寄り添い、長く続く関係を築いていくことへの願いが込められている。

### 1.2 デザイン原則

1. **温かさ（Warmth）**: 飲食の持つ温もり、人と人とのつながりを視覚的に表現する。冷たいデジタル感を避け、手触りのある柔らかな印象を大切にする。

2. **敬意（Respect）**: 飲食人の誇りと情熱に対する敬意を体現するデザイン。押し付けがましさや上から目線を排除し、店舗の魅力を引き立てる控えめな美しさを追求する。

3. **親しみやすさ（Approachability）**: 老若男女、ITリテラシーに関わらず直感的に使える設計。過度な装飾を避け、コンテンツ（ストーリー・写真・メニュー）が主役になるレイアウト。

4. **信頼感（Trust）**: 飲食店が安心して自店の想いを預けられる、消費者が安心して「推し」として関われる、プラットフォームとしての信頼を醸成するデザイン。

## 3. タイポグラフィ

### 3.1 フォントファミリー

**日本語フォント**
```css
font-family: "Noto Sans JP", "Hiragino Kaku Gothic ProN", "Hiragino Sans", Meiryo, sans-serif;
```

**英語・数字フォント**
```css
font-family: "Inter", "Noto Sans JP", sans-serif;
```

**ストーリー本文用（読みやすさ重視）**
```css
font-family: "Noto Serif JP", "Hiragino Mincho ProN", "Yu Mincho", serif;
```

| ウェイト | 値 | 用途 |
|----------|-----|------|
| Regular | 400 | 本文、説明テキスト |

| SemiBold | 600 | 小見出し、ボタンテキスト |
| Bold | 700 | 見出し、強調テキスト |

### 3.4 タイポグラフィガイドライン

- ストーリー本文は明朝体（Noto Serif JP）を使い、読み物としての品格を出す
- UI要素（ボタン、ナビ、ラベル）はゴシック体（Noto Sans JP）で視認性を確保
- 行間は日本語の可読性のため、英語圏のデフォルトより広めに設定（1.7〜1.8）
- 1行あたりの文字数は35〜45文字を目安（ストーリー本文）

## 7. トーン&マナー

### 7.1 言葉遣い

**基本方針**: 温かみがあり、押し付けがましくなく、飲食人をリスペクトする。

| カテゴリ | 推奨 | 避ける |
|---------|------|--------|
| 敬称 | 「お店」「店舗さん」 | 「店」「テナント」 |
| 応援表現 | 「推す」「応援する」「共感する」 | 「評価する」「採点する」「ランキング」 |
| 紹介表現 | 「想いを知る」「ストーリーを読む」 | 「口コミを見る」「レビュー」 |
| 登録促進 | 「推し店に追加しませんか?」 | 「今すぐ登録!」「お見逃しなく!」 |
| エラー | 「うまく読み込めませんでした」 | 「エラーが発生しました」 |
| 空状態 | 「まだストーリーがありません」 | 「データが見つかりません」 |

### 7.2 ビジュアルトーン

- **写真**: 温かみのある暖色系の色味。料理写真は自然光を意識した明るいトーン。人物写真は笑顔や調理中の真剣な表情。
- **イラスト**: MVP では使用しない。使用する場合は手描き風の温かみのあるタッチ。
- **アニメーション**: 控えめで上品。ページ遷移はフェード、共感タップはハートの浮遊アニメーション。過度なアニメーションは避ける。
- **余白**: 十分な余白を取り、コンテンツに呼吸させる。詰め込みすぎない。
- **角丸**: 直角を避け、8〜16pxの角丸で柔らかさを表現。

### 7.3 NGパターン

- 飲食店を「評価」「採点」するニュアンス
- 他店との比較やランキング表示
- 煽り文句や過度なマーケティング表現
- ネガティブな口コミや批判的なコメント機能
- 冷たい・機械的なUI（角張った要素、モノクロ基調）
- 過度なゲーミフィケーション（ポイント、レベル、バッジ）

## 8. 参考サービスからの学び

### 8.1 Wantedly

**参考にする点**
- 企業の「想い」を伝えるストーリー中心の設計思想
- タブ構成による情報の整理（ストーリー/メンバー/企業情報）
- 「共感」ベースのインタラクション（スコアではなく応援）
- ミッション・ビジョンを全面に出したプロフィールページ

**オシドリへの適用**
- 店舗詳細ページをWantedly企業ページのタブ構成で設計
- 「話を聞きに行きたい」に相当する「推す」ボタン
- ストーリーを最上位コンテンツとして設計

### 8.2 Patreon

**参考にする点**
- クリエイターへの「応援」を軸としたプラットフォーム設計
- ファンとクリエイターの双方向の関係構築
- シンプルで温かみのあるUI

**オシドリへの適用**
- 飲食店を「クリエイター」として捉える世界観
- 推し登録をPatreonのフォローに近い体験として設計
- ファンの声（応援）の可視化方法

### 8.3 note

**参考にする点**
- 記事（ストーリー）の読みやすいタイポグラフィ
- スキ（いいね）の温かみのあるアニメーション
- クリエイターの個性が出るプロフィールページ
- 余白を活かしたクリーンなデザイン

**オシドリへの適用**
- ストーリー本文の読みやすさ（明朝体、行間、文字サイズ）
- 共感タップのアニメーション参考
- コンテンツ主体のミニマルなレイアウト
