# オシドリ MVP 要件定義書

## 1. プロジェクト概要

### 1.1 サービス名
**オシドリ** -- 飲食店の「想い」と消費者の「推し」をつなぐプラットフォーム

### 1.2 コンセプト
飲食店が自店のストーリー（創業の想い、食材へのこだわり、料理人の哲学）をAIインタビューを通じて言語化し発信できる場を提供する。消費者は「味」だけでなく「想い」で飲食店を見つけ、共感し、「推し店」として応援できる。

### 1.3 ターゲットユーザー
- **消費者（ファン）**: 20〜40代、飲食店の背景やストーリーに興味がある層
- **店舗オーナー・スタッフ**: 個人経営〜小規模チェーンの飲食店運営者

### 1.4 MVP スコープ
- 消費者向け: 閲覧・検索・推し登録・共感タップ
- 店舗向け: AIインタビューによるストーリー生成・店舗情報管理・簡易ダッシュボード
- 認証: メール認証（MVP）、SNS認証（Google/LINE、MVPまたは次フェーズ）

## 2. 消費者向け画面・機能要件

### 2.1 トップページ

**画面概要**
サービスのエントリーポイント。おすすめストーリーとエリア別の店舗を表示し、消費者が直感的に飲食店のストーリーに出会える。

**機能要件**
| ID | 要件 | 優先度 | 備考 |
|----|------|--------|------|
| C-TOP-01 | ヒーローセクション表示 | 必須 | サービスコンセプトを伝えるキャッチコピーとCTA |
| C-TOP-02 | おすすめストーリーカルーセル | 必須 | 編集部ピックアップまたはPV順で最大10件 |
| C-TOP-03 | エリア別店舗一覧セクション | 必須 | 主要エリアのタブ切り替え表示 |
| C-TOP-04 | カテゴリクイックフィルター | 推奨 | 和食・洋食・中華・カフェなどのアイコンボタン |
| C-TOP-05 | 検索バー | 必須 | フリーワード検索、トップに常時表示 |
| C-TOP-06 | 新着ストーリーセクション | 推奨 | 新しく公開されたストーリーを時系列で表示 |

**表示要素（ストーリーカード）**

- エリア（例：渋谷・恵比寿）

- 推し数（星アイコン + 数値）

### 2.6 共感タップ・感情タグ

**概要**
ストーリーへの共感を「いいね」よりリッチに表現する仕組み。

**共感タップ**
- ストーリー閲覧時に画面下部に固定表示されるボタン
- タップするたびにアニメーション（ハートが浮き上がる等）
- 1ユーザー1ストーリーあたり最大10回タップ可能
- ログイン不要で実行可能（ただし未ログイン時はローカル保存のみ）

**感情タグ**
- ストーリーを読んだ後に選択できる感情ラベル
- 1ユーザー1ストーリーあたり最大3つ選択可能
- ログイン必須

**感情タグ一覧（MVP）**
| タグ名 | 絵文字 | 説明 |
|--------|--------|------|
| 感動した | :sparkles: | ストーリーに心を動かされた |
| 応援したい | :muscle: | この店を応援したいと思った |
| 行ってみたい | :fork_and_knife: | 実際に訪問したくなった |
| 共感する | :handshake: | 想いに共感した |
| 知れてよかった | :bulb: | 新しい発見があった |
| また読みたい | :book: | 他のストーリーも読みたい |

### 3.2 AIインタビュー画面

**画面概要**
AIが対話形式で店舗オーナーにインタビューし、回答をもとにストーリーを自動生成する。チャット型UIで実装。

**インタビューフロー**
1. インタビュー開始（テーマ選択）
2. AIからの質問表示（1問ずつ）
3. ユーザーのテキスト入力（音声入力対応は将来）
4. AIの深掘り質問（回答内容に応じて動的に生成）
5. インタビュー完了（5〜10問程度）
6. ストーリー自動生成・プレビュー
7. 編集・公開

**インタビューテーマ（MVP）**
- 創業ストーリー: お店を始めたきっかけ・想い
- 食材へのこだわり: 仕入れ・産地・選定基準
- 料理哲学: 調理法・味づくりの考え方
- お客様への想い: どんな体験を届けたいか
- スタッフの紹介: チームの魅力・働く環境

**機能要件**
| ID | 要件 | 優先度 | 備考 |
|----|------|--------|------|
| S-AI-01 | テーマ選択画面 | 必須 | カード形式で5テーマ表示 |
| S-AI-02 | チャット型UI | 必須 | AIメッセージと入力欄の交互表示 |
| S-AI-03 | AIによる質問生成 | 必須 | OpenAI API連携 |
| S-AI-04 | 回答に応じた深掘り | 必須 | コンテキストを保持した追加質問 |
| S-AI-05 | インタビュー進捗表示 | 推奨 | プログレスバー（例: 3/8問） |
| S-AI-06 | 途中保存・再開 | 推奨 | インタビューを中断して後日再開 |
| S-AI-07 | ストーリー自動生成 | 必須 | インタビュー完了後に記事形式で生成 |
| S-AI-08 | 生成ストーリープレビュー | 必須 | 公開前にプレビュー確認 |
| S-AI-09 | ストーリー再生成 | 推奨 | トーンや長さを調整して再生成 |

**AI生成ストーリーの仕様**
- 文字数: 800〜2000文字
- トーン: 温かみのある語り口、押し付けがましくない
- 構成: タイトル + 導入 + 本文（2〜3セクション）+ 結びの言葉
- 画像挿入位置の提案（「ここにお店の外観写真を入れると効果的です」等）

| ゲスト | 未ログインユーザー | ストーリー閲覧、検索、共感タップ（ローカル保存のみ） |

### 4.2 店舗アカウント登録フロー

## 6. 非機能要件

### 6.1 パフォーマンス
- ページ読み込み: 3秒以内（LCP）
- API応答: 500ms以内（95パーセンタイル）
- AI生成ストーリー応答: 30秒以内

### 6.2 セキュリティ
- HTTPS必須
- パスワードはbcryptでハッシュ化
- CSRF対策
- XSS対策（入力サニタイゼーション）
- レート制限（API: 100req/min/user）

### 6.3 スケーラビリティ（MVP）
- 同時接続: 100ユーザー
- 店舗数: 100店舗
- ストーリー数: 500件

### 6.4 対応環境
- ブラウザ: Chrome/Safari/Firefox/Edge（最新2バージョン）
- デバイス: スマートフォン（主）、タブレット、デスクトップ
- OS: iOS 15+、Android 12+、Windows 10+、macOS 12+
