### 2026-03-07: UIUX改修計画書作成

**担当業務:** UIUXレビュー（v6.1デプロイ版）を受けて具体的な改修計画を立案

**成果物:** `開発/改修計画/オシドリ_UIUX改修計画_v6_1対応_20260307.md`

**主要決定事項:**

1. **P0-1: 店舗側ダッシュボードのタブ集約（13→6タブ）**
   - 認知負荷54%削減を狙う構造改革
   - 統合後の構成: ホーム/コンテンツ/お客さんとの交流/店舗設定/AI機能/ファンクラブ運営
   - プログレッシブ開示を導入（初回ログイン時は3タブのみ表示、段階的に解放）
   - 工数: 2-3日

2. **P0-2: AIコミュニティマネージャー提案カードをホームに配置**
   - 店主が毎日目にする導線を確保
   - ホーム画面の最上部にAI提案カードを2件表示
   - 提案承認率50%以上を目標に設定
   - 工数: 1日

3. **P1-1: チェックイン=推し登録同時フローの検証・修正**
   - 仕様変更書v6.1の定義に準拠しているか確認
   - デフォルトON実装の検証
   - 工数: 1-2日

4. **P1-2: 気分タグ・パーソナルレコメンドの実装確認**
   - チェックイン時の気分タグ入力UI
   - 体験記録（ダイアリー）の感情タグ入力UI
   - 「今日のおすすめ」パーソナルレコメンド機能
   - 工数: 3-5日

5. **P2-1: オシドリ予報の表現見直し**
   - 「推しスコア排除」の思想に合わせた表現変更
   - 「あなたとの相性」表現に変更
   - 工数: 0.5日

6. **P2-2: ファンレターの目立つ配置**
   - ホーム画面にファンレターセクション追加
   - サイドバーに新着バッジ表示
   - 工数: 0.5日

**実装スケジュール:**
- Phase 1 (Day 1-3): P0項目（タブ集約 + AI提案カード）
- Phase 2 (Day 4-10): P1項目（チェックインフロー + 気分タグ）
- Phase 3 (Day 11-30): P2項目（表現見直し + ファンレター配置）

**期待効果:**
- 店主の認知負荷54%削減
- AIコミュニティマネージャー利用率向上
- チェックイン→推し登録コンバージョン率+30%
- 共感マッチング精度向上

**KPI設定:**
- タブ数: 13→6 (54%削減)
- ダッシュボード滞在時間: +30%
- AI提案承認率: 50%以上
- チェックイン→推し登録CVR: +30%
- 気分タグ入力率: 60%以上

**学び:**
- UIUXレビューで明確になった課題を、優先度と工数で構造化することで実行可能な計画に落とし込んだ
- プログレッシブ開示の導入により、初心者店主でも段階的に機能を習得できる設計を考案
- AIコミュニティマネージャーの「毎日目に入る導線」の重要性を認識

### 2026-03-07: トップページ改善の実装戦略立案

**タスク:** CMOひなた・CCOみずきの提案を踏まえたトップページ改善の技術的実装可能性評価

**成果物:** `開発/改修計画/オシドリ_トップページ改善実装ロードマップ_20260307.md`

**現状確認結果:**
1. ホームページ実装状況
   - ファイル: `src/app/(consumer)/home/page.tsx` (852行)
   - 既存セクション: ヒーロー、マイコレクション、パーソナルレコメンド、推し店更新、ピックアップ、旬のおすすめ、相性予報、テーマコレクション、週間ストーリー、気分検索、駅検索、オシドリとは、オーナーCTA
   - 既存コンポーネント活用可能: SmallCard、HorizontalCard、ForecastBadge

2. パーソナルレコメンドAPI実装済み
   - ファイル: `src/app/api/recommendations/personal/route.ts`
   - 気分タグベースマッチング実装済み
   - デフォルトレコメンド（人気店）実装済み

3. 既存コンポーネント確認
   - `src/components/mood-tag-selector.tsx` 存在
   - `src/components/checkin-oshi-flow.tsx` 存在
   - `src/components/ai-cm-proposal-card.tsx` 存在

**主要決定事項:**

**MVP（即座実装）:**
1. ストーリーカード強化（hook_sentence活用）
2. メニュー写真ギャラリー追加
3. チェックイン履歴の可視化
4. 推し店更新通知の実装

**Phase 1（PoC期間中）:**
1. 気分タグフィルタ強化
2. 旬のおすすめアルゴリズム改善
3. ストーリー閲覧履歴ベースレコメンド

**Phase 2（成長期）:**
1. 協調フィルタリング
2. 店舗間類似度計算
3. 地図連動UI

**技術的実装可能性評価:**
- 既存コンポーネント活用度: 80%（SmallCard/HorizontalCard/ForecastBadge）
- 新規開発要素: メニューギャラリー、チェックイン履歴UI、通知システム
- パーソナルレコメンドAPI: 実装済み、拡張容易
- データ要件: 既存DBスキーマで対応可能（visits, oshi_shops, consumer_mood_preferences）

**リスク評価:**
- パフォーマンス: 画像遅延読み込み必須（Next.js Image最適化活用）
- 初期データ不足: フォールバック設計済み（人気店表示）
- 複雑度増加: プログレッシブ開示で段階的公開

**実装優先順位の論理:**
1. MVP: 既存データ活用、低開発コスト、高ユーザー価値
2. Phase 1: データ蓄積後に精度向上
3. Phase 2: ユーザー数増加後に協調フィルタリング有効化

**次のアクション:**
- 詳細実装ロードマップ作成
- A/Bテスト設計書作成
- 開発チームと工数見積もり調整

**学び:**
- 既存実装の把握により、CMO/CCO提案の80%が既存コンポーネント活用で実現可能と判明
- パーソナルレコメンドAPIが既に実装済みで、追加開発コストが想定より低い
- データ駆動型改善には初期フォールバック設計が不可欠
