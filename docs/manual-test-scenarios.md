# 手動テストシナリオ（Phase 1-3）

PoC実施前にクリティカルパスを手動確認するためのチェックリスト。

---

## 1-3-1: 一般ユーザー初回体験

**前提:** 新規ユーザー（未登録状態）

| # | 操作 | 確認ポイント | 結果 |
|---|------|-------------|------|
| 1 | `/lp` にアクセス | LP が表示される。CTA ボタンが目立つ | |
| 2 | 「始める」ボタンをタップ | ログインページに遷移する | |
| 3 | Google/メール で登録 | Supabase Auth が動作し認証完了 | |
| 4 | ニックネーム設定画面 | 入力→送信でオンボーディング完了 | |
| 5 | ホーム画面に遷移 | ピックアップ店舗が表示される | |
| 6 | 「さがす」タブで店舗検索 | 店舗一覧が表示、フィルタが動作する | |
| 7 | 「メニューから探す」タブに切替 | メニュー検索が動作する | |
| 8 | 店舗詳細ページを閲覧 | ストーリー・メニュー・ファンクラブが表示 | |
| 9 | 推し店登録ボタンをタップ | 「推し！」状態になり、推し店一覧に追加 | |
| 10 | 推し店を解除 | 「推し！」解除→一覧から消える | |

## 1-3-2: 店舗オーナー初回体験

**前提:** 管理者が事前に店舗申請を承認

| # | 操作 | 確認ポイント | 結果 |
|---|------|-------------|------|
| 1 | `/for-shops` にアクセス | 飲食店向けLPが表示される | |
| 2 | 店舗申請フォーム送信 | 申請完了メッセージが表示される | |
| 3 | 管理者が `/admin` で承認 | ステータスが「承認済み」に変わる | |
| 4 | オーナーがダッシュボードへ | プロフィール編集画面が表示される | |
| 5 | 店舗プロフィール編集 | 店名、説明、画像を変更→保存成功 | |
| 6 | メニュー登録 | メニュー名、価格、写真→保存成功 | |
| 7 | AIインタビュー開始 | ナオ（AI）が挨拶メッセージを返す | |
| 8 | 5往復会話 | 質問が自然、フェーズ遷移が適切 | |
| 9 | インタビュー完了 | ストーリー自動生成→プレビュー表示 | |
| 10 | ストーリー公開 | 消費者側で店舗詳細にストーリー表示 | |

## 1-3-4: 来店〜再訪導線（CRM機能）

**前提:** ログイン済みユーザー、推し店1件以上

| # | 操作 | 確認ポイント | 結果 |
|---|------|-------------|------|
| 1 | QRチェックイン | 店舗QRコード読取→来店記録作成 | |
| 2 | 気分タグ選択 | 複数タグ選択可、保存される | |
| 3 | メモ入力 | 500文字以内で保存、超過は切り詰め | |
| 4 | エンパシー送信 | ストーリーに共感タップ→カウント増加 | |
| 5 | ファンレター送信 | テキスト入力→送信→オーナーに届く | |
| 6 | 推し店一覧 | 登録済み推し店が正しく表示 | |
| 7 | 日記（ダイアリー） | 来店履歴が時系列で表示 | |
| 8 | 再訪促進通知 | Supply Flash が推し店からの通知として表示 | |
