## 1. プロダクト概要

### コンセプト
飲食店の「想い」と消費者の「推し」をつなぐコミュニティハブ。
AIインタビューで店主の想いを引き出し、ストーリーとして発信。消費者は共感・推し登録を通じてお店とつながる。

### ターゲットユーザー
- **生活者（消費者）**: 食に関心があり、お店の「想い」に共感したい人
- **飲食店オーナー**: 自店の魅力を言語化し、ファンとの関係を深めたい人

### 技術スタック
| 領域 | 技術 |
|------|------|
| フレームワーク | Next.js 16.1.6 (App Router) + TypeScript |
| UI | Tailwind CSS 4 + shadcn/ui (Radix UI) |
| データベース | Supabase (PostgreSQL + Auth + Storage) |
| AI | OpenAI GPT-4o (Vercel AI SDK) |
| ホスティング | Vercel (Hobby Plan) |
| 分析 | PostHog |
| 地図 | Google Maps API |

### 4-3. RLSポリシー方針
- **公開データ**（shops, stories, menus）: 全員READ可

- **店舗所有データ**（dashboard系）: owner_idが一致する店舗のみ
- **管理データ**（admin系）: is_admin=true のみ

## 5. AI機能仕様

### 5-1. AIインタビューシステム

**ペルソナ:** 「ナオ」── 共感力の高い食の編集者
**モデル:** OpenAI GPT-4o（ストリーミングレスポンス）

#### インタビュータイプ（4種）
| タイプ | フェーズ数 | 目的 |
|--------|----------|------|
| initial_interview | 6 | 初回取材（原点→こだわり→一品→常連→未来） |
| monthly_update | 3 | 月次更新（近況→エピソード→締め） |
| menu_addition | 3 | 新メニュー取材 |
| seasonal_menu | 3 | 季節メニュー取材 |

#### 初回インタビューのフェーズ
1. **warmup** — アイスブレイク（お店の雰囲気を聞く）
2. **origin** — 原点・きっかけ（なぜこのお店を？）
3. **kodawari** — こだわり（食材・技術・哲学）
4. **menu_story** — 一品の物語（看板メニューの裏話）
5. **regulars** — 常連さんとの関係
6. **future** — 未来への想い

#### 自動生成物
- **ストーリー**: タイトル + 本文（Markdown） + 要約 + キー引用文
- **7テーマスコア**: origin / food_craft / hospitality / community / personality / local_connection / vision（各0-10）
- **構造化タグ**: kodawari[] / personality[] / scene[] / atmosphere[]
- **メニュー情報**: 名前 / 説明 / こだわりポイント / 食べ方ヒント
- **写真リクエスト**: 撮影指示（被写体・説明・優先度）
- **L1/L2キャッチコピー**: プライマリ + 代替2種 + ハイライト + フック文

#### エンゲージメントコンテキスト自動注入
次回インタビュー時に、前回の共感データ（高共感テーマ・感情タグ傾向）をシステムプロンプトに注入し、より深い取材を実現。

### 5-2. おしどり予報スコアリング
- **基本スコア**: story_themes × user_value_vectors のJaccard係数
- **エンゲージメントボーナス**: oshi×0.3 + empathy×0.1（最大+2）
- **表示**: 5段階ドット表示（forecast-dots）

## 7. 認証・権限システム

### 認証方式
- Email/Password（Supabase Auth）
- Google OAuth
- オンボーディング: ログイン後にニックネーム登録

### 3層権限モデル
| ロール | アクセス範囲 |
|--------|------------|
| consumer | 消費者サイト全般 |
| shop_owner | 消費者サイト + 店舗ダッシュボード |
| admin | 全ページ（管理画面含む） |

### 権限フロー
1. 新規登録 → `consumer` ロール自動付与
2. `/apply-shop-owner` から店舗権限申請
3. admin が `/admin/applications` で審査 → 承認で `shop_owner` に昇格
4. admin権限はDBマイグレーションで手動付与
